# Task Plan Assistant

本 Skill 帮助你在执行复杂任务时有效管理上下文，通过任务规划 MCP 工具实现：

## 核心功能

### 规划上下文管理
使用 `create_context` 工具为每个任务创建独立规划上下文，支持：
- 任务名称和角色定义
- 父子上下文层级关系
- Agent 环境信息配置

### 计划反馈追踪
使用 `save_important_plan_feedback_memory` 工具保存关键计划决策：
- 记录计划与反馈对话历史
- 关联元数据和 Plan ID
- 保持决策可追溯性

### 工具执行监控
使用 `save_tool_execution_feedback_memory` 工具记录执行细节：
- 工具名称、成功状态、执行时间
- 输入输出数据
- Token 消耗统计

### 记忆压缩
使用 `compress_history_messages` 工具：
- 压缩历史消息减少上下文长度
- 保持关键信息完整性

## 使用场景

- **复杂多步骤任务**：创建独立上下文避免信息混淆
- **迭代开发过程**：记录每个迭代的计划和反馈
- **工具调用优化**：监控 Token 使用和执行效率
- **知识沉淀**：长期记忆检索和复用

## 快速开始

```
创建一个新的规划上下文用于处理用户需求分析任务
保存当前计划讨论的关键决策
记录工具执行结果
压缩历史消息减少上下文长度
```
